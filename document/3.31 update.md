## 二、逐条改动说明

### 1. 全局异常处理（@RestControllerAdvice）

**新增文件**：`config/GlobalExceptionHandler.java`

- 统一处理 `MethodArgumentNotValidException`、`BindException` → 400
- 统一处理 `BadCredentialsException` → 401
- 统一处理 `DisabledException` → 403
- 统一处理 `IllegalArgumentException` → 409
- 统一处理其他 `Exception` → 500

**效果**：Controller 不再需要 try-catch，直接抛出异常即可，返回格式统一。

---

### 2. 配置外置化

**修改文件**：`src/main/resources/application.yaml`

新增/扩展配置：
```yaml
app:
  jwt:
    expiration-seconds: 3600
  token:
    redis-prefix: "login:token:"
  cors:
    allowed-origins:
      - "http://localhost:5173"
```

**新增文件**：`config/AppProperties.java`

- 使用 `@ConfigurationProperties(prefix = "app")` 读取配置
- 提供 `Jwt`、`Token`、`Cors` 配置类
- `SecurityConfig` 的 CORS 改为从 `AppProperties` 读取
- `TokenServiceImpl` 的 JWT 密钥、过期时间、Redis 前缀均从配置读取

---

### 3. Service 层拆分

| 服务 | 职责 |
|------|------|
| `AuthService` | 登录、登出、当前会话查询 |
| `RegisterService` | 注册（用户名校验、密码加密、落库） |
| `UserService` | 用户查询（`getByUsername`），继承 `IService<SysUsers>` |
| `TokenService` | Token 生成、解析、Redis 缓存与删除 |

**删除**：`LoginService`、`LoginServiceImpl`，由上述服务替代。

---

### 4. 抽取 Token 组件

**新增文件**：
- `service/TokenService.java`（接口）
- `service/impl/TokenServiceImpl.java`（实现）

**职责**：
- `generateToken(username)`：生成 JWT
- `parseUsername(token)`：解析 JWT 获取用户名
- `cacheToken(username, token)`：写入 Redis
- `getCachedToken(username)`：从 Redis 读取
- `deleteToken(username)`：从 Redis 删除
- `validate(username, token)`：校验 token 是否有效

**效果**：JWT 与 Redis 逻辑集中管理，`JwtUtil` 静态工具类不再使用，全部由 `TokenService` 接管。

---

### 5. SessionController 精简

**修改文件**：`controller/SessionController.java`

- 移除所有 try-catch，依赖全局异常处理
- 改为注入 `AuthService`、`RegisterService`
- 接口逻辑只做参数接收、调用服务、返回结果

---

### 6. 实体与包结构

- 确认仅保留 `SysUsers` 实体，无 `sysUsers` 重复引用
- 所有引用已统一为 `org.tonyqwe.cinemaweb.domain.entity.SysUsers`

---

## 三、当前包结构与依赖关系

```
controller/
  SessionController      → AuthService, RegisterService

service/
  AuthService            → UserService, TokenService, PasswordEncoder
  RegisterService        → UserService, PasswordEncoder
  UserService            → UserMapper (IService<SysUsers>)
  TokenService           → StringRedisTemplate, AppProperties

config/
  GlobalExceptionHandler
  AppProperties          → application.yaml (app.*)
  SecurityConfig         → JwtAuthenticationFilter, AppProperties
  PasswordEncoderConfig
  JwtAuthenticationFilter → AuthService, PermissionService
```

---

## 四、后续扩展建议

1. **新业务模块**：按 `controller/foo`、`service/foo`、`mapper` 拆分，沿用现有分层
2. **新接口**：直接抛业务异常，由 `GlobalExceptionHandler` 统一处理
3. **新配置**：在 `application.yaml` 的 `app` 下扩展，并在 `AppProperties` 中增加对应属性
